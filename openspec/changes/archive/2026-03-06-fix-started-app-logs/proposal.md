## Why

When users start an application from the gallery/app list, the log terminal/console does not display the logs for the started application. This makes it difficult for users to debug or monitor the status of the launched applications, preventing them from seeing errors or output that the application might generate.

## What Changes

- Fix the log terminal to correctly stream and display standard output/error from launched applications.
- Ensure that the UI properly connects the started app's process output to the frontend terminal component.
- Verify that the terminal updates in real-time as new logs are generated by the app.

## Capabilities

### New Capabilities

- `app-log-streaming`: Fix the backend-to-frontend streaming of logs from a started application process to the frontend terminal UI.

### Modified Capabilities

- `gallery-log-inline-panel`: Ensure that the existing log panel correctly receives and displays the log streams from started apps.

## Impact

- Frontend terminal or log view components (`FolderCard.tsx` or related log components).
- Backend process management and logging stream handlers.
