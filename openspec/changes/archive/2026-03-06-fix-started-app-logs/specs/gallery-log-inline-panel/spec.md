## MODIFIED Requirements

### Requirement: Log Panel Renders Inline For Non-Folder Items

The system SHALL display the log panel as an inline right-side panel within the expanded item overlay for non-folder items, matching the same side-by-side pattern used for folder items. The system MUST render the actual log output continuously generated by the item's child process.

#### Scenario: User opens log terminal for a non-folder item

- **WHEN** the user toggles the log terminal button on a non-folder item in the gallery view
- **THEN** the item's overlay expands horizontally to show the log panel to the right of the item content
- **AND** the console output is visible and dynamically rendering streaming process lines within the log panel

#### Scenario: Console output is available

- **WHEN** the log panel is visible for any item (folder or non-folder)
- **THEN** the console area SHALL display all incoming log output lines from connected streams and not remain empty if valid streaming data arrives
