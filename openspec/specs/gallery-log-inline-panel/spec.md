## ADDED Requirements

### Requirement: Log Panel Renders Inline Within Folder Overlay

The system SHALL display the log panel as an inline side panel within the same overlay container as the folder popup, using a horizontal flex layout, so that folder content and logs appear side-by-side at the same z-plane.

#### Scenario: User opens log terminal for a folder item

- **WHEN** the user toggles the log terminal button on a folder item in the gallery view
- **THEN** the log panel appears to the right of the folder content within the same overlay container
- **AND** no new overlay or stacking context is created above the folder popup
- **AND** the gallery grid layout remains unchanged (no inline expansion)

#### Scenario: User closes log terminal for a folder item

- **WHEN** the user toggles the log terminal button again on an open folder item
- **THEN** the log panel is hidden and the overlay container returns to showing only the folder content
- **AND** the overlay container does not change its z-level or position

### Requirement: Log Panel Renders Inline For Non-Folder Items

The system SHALL display the log panel as an inline right-side panel within the expanded item overlay for non-folder items, matching the same side-by-side pattern used for folder items. The system MUST render the actual log output continuously generated by the item's child process.

#### Scenario: User opens log terminal for a non-folder item

- **WHEN** the user toggles the log terminal button on a non-folder item in the gallery view
- **THEN** the item's overlay expands horizontally to show the log panel to the right of the item content
- **AND** the console output is visible and dynamically rendering streaming process lines within the log panel

#### Scenario: Console output is available

- **WHEN** the log panel is visible for any item (folder or non-folder)
- **THEN** the console area SHALL display all incoming log output lines from connected streams and not remain empty if valid streaming data arrives

### Requirement: Log Panel Does Not Overflow Viewport

The system SHALL ensure the combined width of the folder panel and log panel does not cause horizontal scroll or viewport overflow.

#### Scenario: Combined overlay width exceeds viewport

- **WHEN** the folder overlay plus the log panel width would exceed the viewport width
- **THEN** the overlay container SHALL constrain itself within the viewport bounds (e.g., via max-width or overflow: hidden)
